OpenVPN Windows Server

If you want to configure a Windows computer to act as a VPN server, using the free community edition of OpenVPN, then you must carry out 17 steps.

You can watch a demonstration of the 17 steps in a YouTube video:

https://www.youtube.com/watch?v=K4kAR3M0XHU

1. Open port udp/1194 by creating a new input rule in Windows Defender Firewall with Advanced Security.

2. Install OpenVPN and EasyRSA from https://openvpn.net/community-downloads.

3. Install EasyTLS from https://github.com/TinCanTech/easy-tls/releases. Copy the 8 extracted files into the same folder as easyrsa.

4. Initialize Public Key Infrastructure using the EasyRSA-Start batch file and the easyrsa script.

    cd C:\Program Files\OpenVPN\easy-rsa

    copy vars.example vars

    notepad vars

    EasyRSA-Start.bat

    ./easyrsa init-pki

5. Create a Certificate Authority (CA) using easyrsa.

    ./easyrsa build-ca nopass

6. Build a server key and certificate for server "moon" using easyrsa.

    ./easyrsa build-server-full moon nopass

7. Build a client key and certificate for client "carol" using easyrsa.

    ./easyrsa build-client-full carol nopass

8. Create Diffie-Hellman parameters using easyrsa.

    ./easyrsa gen-dh

9. Create TLS Crypt v2 keys using the easytls script.

    ./easytls init-tls

You may get a message, Error: Unsupported OpenSSL version: 3.4. In this case, do:

    notepad easytls

Find the block:

	case "${openssl_version}" in
		3.0|3.1) : ;;
		1.1) : ;;
		*) die "Unsupported OpenSSL version: ${openssl_version}" ;;
	esac

Add the case of OpenSSL later versions:

	case "${openssl_version}" in
		3.0|3.1|3.2|3.3|3.4) : ;;
		1.1) : ;;
		*) die "Unsupported OpenSSL version: ${openssl_version}" ;;
	esac

Save the file, and close Notepad. Then re-run the command:

    ./easytls init-tls

    ./easytls build-tls-crypt-v2-server moon

    ./easytls build-tls-crypt-v2-client moon carol

    ./easytls inline-tls-crypt-v2 carol

The inline TLS crypt v2 file is created at C:/Program Files/OpenVPN/easy-rsa/pki/easytls/carol.inline. The inline file is for eventual incorporation into your client configuration .ovpn file.

10. Create the server configuration for server "moon", for example:

    port 1194
    proto udp
    dev tun
    topology subnet
    cipher AES-256-GCM
    ca "C:\\Program Files\\OpenVPN\\easy-rsa\\pki\\ca.crt"
    cert "C:\\Program Files\\OpenVPN\\easy-rsa\\pki\\issued\\moon.crt"
    key "C:\\Program Files\\OpenVPN\\easy-rsa\\pki\\private\\moon.key"
    dh "C:\\Program Files\\OpenVPN\\easy-rsa\\pki\\dh.pem"
    tls-crypt-v2 "C:\\Program Files\\OpenVPN\\easy-rsa\\pki\\easytls\\moon-tls-crypt-v2.key"
    server 10.8.0.0 255.255.255.0
    ifconfig-pool-persist ipp.txt
    push "redirect-gateway def1"
    push "dhcp-option DNS 1.1.1.1"
    push "dhcp-option DNS 1.0.0.1"
    push "block-outside-dns"
    keepalive 10 60
    persist-key
    persist-tun
    explicit-exit-notify 1
    verb 3

11. Create the client configuration for "carol", incorporating the inline file created by the easytls script, e.g.:

    client
    dev tun
    proto udp
    cipher AES-256-GCM
    remote 123.123.123.123 1194
    resolv-retry infinite
    nobind
    persist-key
    persist-tun

    (Incorporate C:\Program Files\OpenVPN\easy-rsa\pki\easytls\carol.inline here)

    remote-cert-tls server
    verb 3

12. Enable forwarding on your OpenVPN TAP-Windows6 interface using the netsh command in an elevated command prompt.

    netsh int ipv4 show int

Replace xx by your OpenVPN TAP-Windows6 interface number in these commands:

    netsh int ipv4 show int xx | findstr "Forwarding"

    netsh int ipv4 set int xx Forwarding="enabled"

    netsh int ipv4 show int xx | findstr "Forwarding"

13. Enable routing using the Registry Editor to set HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\Tcpip\Parameters\IPEnableRouter to value 1.

14. Start the service Routing and Remote Access and set it to automatic start; stop OpenVPN Interactive Service and set it to manual start; and restart OpenVPNService.

15. Allow Internet Connection Sharing. On your computer's main interface's Sharing tab, check the box to Allow other network users to connect through this computer's Internet connection, and specifically allow the OpenVPN TAP-Windows6 to share it.

16. Disable and re-enable the OpenVPN TAP-Windows6 interface.

17. Check that your computer is listening on port udp/1194, and check the log for confirmation and the OpenVPN initialization sequences has completed.

If you experience any problems, try rebooting your computer.

The Windows computer in the video is located in a data center. In a home network, you would need to open your router firewall, port forward OpenVPN traffic from your router to your PC, and configure the PC power plan so that your PC does not sleep or hibernate. Using a home PC will work only if your ISP gives your home a unique IP address (i.e. no Carrier Grade NAT).

If your computer operating system is Windows Server, remember that IE Enhanced Security Configuration must be set to off to use the browser.
