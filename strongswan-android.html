Configure strongSwan server for Android clients

Server

1. Allow forwarding in the Linux kernel

Edit /etc/sysctl.conf.

Uncomment the line:

net.ipv4.ip_forward=1

Save the file and activate the change:

sysctl -p

2. Add iptables rules

Open iptables for IPsec input:

iptables -A INPUT -p udp --dport 500 -j ACCEPT

iptables -A INPUT -p udp --dport 4500 -j ACCEPT

Assuming your main interface is named ens3, masquerade the IP address of the output packets:

iptables -t nat -A POSTROUTING -s 10.3.0.0/16 -o ens3 -j MASQUERADE

Assuming you have previously installed and configured iptables-persistent, persist the revised set of rules as they now stand:

dpkg-reconfigure iptables-persistent

3. Install strongSwan on server

Install strongSwan:

apt update && apt upgrade -y && apt autoremove -y

apt install -y strongswan strongswan-swanctl strongswan-pki libstrongswan libstrongswan-standard-plugins libstrongswan-extra-plugins libcharon-extra-plugins libcharon-extauth-plugins charon-systemd libtss2-tcti-tabrmd0

4. Create keys and certificates

Make a directory for certificate requests:

mkdir /etc/swanctl/req

Make a self-signed CA certificate. Note that Android does not accept key types such as ed25519. For maximum cross-platform compatibility, use key type rsa:

pki --gen --type rsa --size 3072 --outform pem > /etc/swanctl/private/strongswanKey.pem

pki --self --ca --lifetime 3652 --in /etc/swanctl/private/strongswanKey.pem --dn "C=CH, O=strongSwan, CN=strongSwan Root CA" --outform pem > /etc/swanctl/x509ca/strongswanCert.pem

Generate a server certificate identified by IP address. Change 123.45.67.89 to your actual server IP address. Make other changes to the certificate country and organization as desired:

pki --gen --type rsa --size 3072 --outform pem > /etc/swanctl/private/123.45.67.89.pem

pki --req --type priv --in /etc/swanctl/private/123.45.67.89.pem --dn "C=CH, O=strongswan, CN=123.45.67.89" --san 123.45.67.89 --outform pem > /etc/swanctl/req/123.45.67.89.pem

pki --issue --cacert /etc/swanctl/x509ca/strongswanCert.pem --cakey /etc/swanctl/private/strongswanKey.pem --type pkcs10 --in /etc/swanctl/req/123.45.67.89.pem --flag serverAuth --flag ikeIntermediate --lifetime 1826 --outform pem > /etc/swanctl/x509/123.45.67.89.pem

Generate a client certificate identified by your client's name. Change alice to your desired name:

pki --gen --type rsa --size 3072 --outform pem > /etc/swanctl/private/aliceKey.pem

pki --req --type priv --in /etc/swanctl/private/aliceKey.pem --dn "C=CH, O=strongswan, CN=alice" --san alice --outform pem > /etc/swanctl/req/aliceReq.pem

pki --issue --cacert /etc/swanctl/x509ca/strongswanCert.pem --cakey /etc/swanctl/private/strongswanKey.pem --type pkcs10 --in /etc/swanctl/req/aliceReq.pem --flag clientAuth --lifetime 1826 --outform pem > /etc/swanctl/x509/aliceCert.pem

For Android, you must export your .p12 file with the option for legacy encryption, as Android cannot handle newer encryption:

openssl pkcs12 -export -legacy -inkey /etc/swanctl/private/aliceKey.pem -in /etc/swanctl/x509/aliceCert.pem -certfile /etc/swanctl/x509ca/strongswanCert.pem -out ~/alice-legacy.p12

5. Create strongSwan configuration

Edit your swanctl configuration file /etc/swanctl/swanctl.conf using the recommended VICI format:

connections {
    rw {
        version = 2
        pools = rw-pool
        local {
            auth = pubkey
            certs = 123.45.67.89.pem
            id = 123.45.67.89
        }
        remote {
            auth = pubkey
        }
        children {
            rw-child {
                local_ts = 0.0.0.0/0
            }
        }
    }
}
pools {
    rw-pool {
        addrs = 10.3.0.0/16
        dns = 1.1.1.1, 1.0.0.1
    }
}

Save the swanctl configuration.

6. Run strongSwan on server

Restart strongSwan:

systemctl restart strongswan

exit

Client

1. Securely copy the legacy-encrypted .p12 file from your server to your Android device

Either transfer from server to PC then from PC to Android, or use Termius on the Android device.

2. Import p12 file to Android client

This varies a bit from version of Android to version of Android. On Android 13 the option to import a user certificate is under Settings > Unlock and security > More security settings > Encryption and credentials > Install a certificate > VPN & app user certificate.

3. Install and configure strongSwan on Android client

From the Google Play store, install strongSwan VPN Client by strongSwan Project.

4. Add a VPN profile with IKEv2 certificate authentication

Select your imported .p12 user certificate. If desired, select advanced settings (e.g. to block traffic that is not going to the VPN). Save your new VPN profile.

5. Connect.

On the screen with your VPN profile(s) on it, tap your desired row to connect the client to the server.
