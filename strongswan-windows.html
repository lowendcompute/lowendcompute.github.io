strongSwan IKEv2 IPsec server with Windows user-password client authentication

1. Enable forwarding in Linux kernel

Open Windows PowerShell.

SSH into your server as root, e.g.:

ssh root@123.123.123.123

Edit the system control configuration file:

vi /etc/sysctl.conf

Uncomment the line to allow forwarding:

net.ipv4.ip_forward=1

Save the file.

Activate the change:

sysctl -p

2. Open firewall for IPsec

Add iptables rules to allow input on ports udp/500 and udp/4500:

iptables -A INPUT -p udp --dport 500 -j ACCEPT

iptables -A INPUT -p udp --dport 4500 -j ACCEPT

Masquerade the source address on outbound packets:

iptables -t nat -A POSTROUTING -s 10.3.0.0/16 -o ens3 -j MASQUERADE

Persist the new set of firewall rules (assuming you have previously installed iptables-persistent):

dpkg-reconfigure iptables-persistent

3. Install packages

Update your system:

apt update && apt upgrade -y && apt autoremove -y

Install strongSwan and related packages:

apt install -y strongswan strongswan-swanctl strongswan-pki libstrongswan libstrongswan-standard-plugins libstrongswan-extra-plugins libcharon-extra-plugins libcharon-extauth-plugins charon-systemd libtss2-tcti-tabrmd0

4. Generate CA key and certificate

Use the strongSwan pki tool to create CA key and certificate. Use key type RSA for compatibility with Windows (and other platforms):

pki --gen --type rsa --outform pem >/etc/swanctl/private/strongSwanKey.pem

pki --self --ca --lifetime 3652 --in /etc/swanctl/private/strongSwanKey.pem --dn "C=CH, O=strongSwan, CN=strongSwan Root CA" --outform pem > /etc/swanctl/x509ca/strongSwanCert.pem

Convert the CA certificate to binary for subsequent download and import to Windows:

openssl x509 -in /etc/swanctl/x509ca/strongSwanCert.pem -out ~/strongSwanCert.cer -outform der

5. Generate server key and certificate

Create a directory for certificate requests:

mkdir /etc/swanctl/req

Create the server key and certificate, replacing 123.123.123.123 in the examples by your actual server IP address:

pki --gen --type rsa --outform pem > /etc/swanctl/private/123.123.123.123.pem

pki --req --type rsa --in /etc/swanctl/private/123.123.123.123.pem --dn "C=CH, O=strongSwan, CN=123.123.123.123" --san 123.123.123.123 --outform pem > /etc/swanctl/req/123.123.123.123.pem

Issue the certificate with the flags required on the server certificate by Windows:

pki --issue --cacert /etc/swanctl/x509ca/strongSwanCert.pem --cakey /etc/swanctl/private/strongSwanKey.pem --type pkcs10 --in /etc/swanctl/req/123.123.123.123.pem --lifetime 730 --flag serverAuth --flag ikeIntermediate --outform pem > /etc/swanctl/x509/123.123.123.123.pem

6. Generate random user password

< /dev/urandom tr -dc a-z0-9 | head -c${1:-16}; echo ''

Example:

pkv3yz1nnf956g29

7. Configure server

Edit the VICI-format configuration file:

vi /etc/swanctl/swanctl.conf

Delete the existing contents.

Insert new contents, using this as a model:

    connections {
        rw {
            version = 2
            pools = rw-pool-v4, rw-pool-v6
            local {
                auth = pubkey
                certs = 123.123.123.123.pem
                id = 123.123.123.123
            }
            send_cert = always
            remote {
                auth = eap-mschapv2
                eap_id = %any
            }
            children {
                rw-child {
                    local_ts = 0.0.0.0/0,::/0
                }
            }
        }
    }
    pools {
        rw-pool-v4 {
            addrs = 10.3.0.0/16
            dns = 1.1.1.1, 1.0.0.1
        }
        rw-pool-v6 {
            addrs = fec3::/120
            dns = 2606:4700:4700::1111, 2606:4700:4700::1001
       }
    }
    secrets {
        eap-emily4 {
            id = emily4
            secret = pkv3yz1nnf956g29
        }
    }

Save the swanctl configuration file.

8. Run strongSwan on server

Stop and disable strongswan-starter:

systemctl stop strongswan-starter

systemctl disable strongswan-starter

Restart strongSwan:

systemctl restart strongswan

systemctl status strongswan

journalctl -u strongswan

exit

9. Download CA certificate from server to Windows client

Note that CA certificates do not need to be kept secret. You can allow anyone to have them.

Open Windows PowerShell.

Issue the command:

scp root@123.123.123.123:strongSwanCert.cer Downloads

10. Import the CA certificate to Windows

Make a note of your Windows user name from the prompt, e.g. C:\Users\<your-win-user-name>.

Close Windows PowerShell as a regular user.

Open Windows PowerShell with Run as Administrator.

Issue the command:

Import-Certificate -FilePath C:\Users\\Downloads\strongSwanCert.cer -CertStoreLocation Cert:\LocalMachine\Root

11. Allow stronger encryption than Windows default

Still in Windows PowerShell opened with Run as Administrator, insert a new registry entry to allow stronger encryption than Windows allows by default:

New-ItemProperty -Path 'HKLM:\SYSTEM\CurrentControlSet\Services\RasMan\Parameters' -Name NegotiateDH2048_AES256 -PropertyType DWord -Value 1

Close Windows PowerShell that was opened with Run as Administrator.

12. Add IKEv2 VPN connection

Open Windows PowerShell as a regular user.

Issue the command:

Add-VpnConnection -Name 'Emily VPN' -ServerAddress '123.123.123.123' -TunnelType IKEv2 -AuthenticationMethod Eap -EncryptionLevel Required -RememberCredential -PassThru

13. Configure IKEv2 parameters

Issue the command:

Set-VpnConnectionIPsecConfiguration -Name 'Emily VPN' -AuthenticationTransformConstants GCMAES128 -CipherTransformConstants GCMAES128 -EncryptionMethod AES256 -IntegrityCheckMethod SHA256 -PfsGroup None -DHGroup Group14 -PassThru -Force

Close Windows PowerShell.

14. Connect client to server

Open Settings > Network & internet > VPN.

Locate the row for Emily VPN.

Press Connect.

Type User name emily4

Type Password pkv3yz1nnf956g29

Click OK.
