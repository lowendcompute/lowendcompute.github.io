strongSwan server with Let's Encrypt server certificate and iOS user-password client authentication

1. Add DNS "A" record

Add a DNS "A" record pointing from vpntest3.mydomain.mytld to your server IP address.

(Here as elsewhere in this tutorial, change vpntest3.mydomain.mytld and other values to your actual desired values.)

2. Allow forwarding in the Linux kernel

SSH into server as root:

    ssh root@vpntest3.mydomain.mytld

Edit /etc/sysctl.conf:

    vi /etc/sysctl.conf

Uncomment the line:

    net.ipv4.ip_forward=1

Save the file. Activate the change:

    sysctl -p

3. Add iptables rules

Open port tcp/80 to allow certbot to get in:

    iptables -A INPUT -p tcp --dport 80 -j ACCEPT

Open ports udp/500 and udp/4500 for IPsec input:

    iptables -A INPUT -p udp --dport 500 -j ACCEPT

    iptables -A INPUT -p udp --dport 4500 -j ACCEPT

Assuming your main interface is named ens3, masquerade the IP address of the output packets:

    iptables -t nat -A POSTROUTING -s 10.3.0.0/16 -o ens3 -j MASQUERADE

Assuming you have previously installed and configured iptables-persistent, persist the revised set of rules as they now stand:

    dpkg-reconfigure iptables-persistent

4. Install strongSwan on server

Install strongSwan:

    apt update && apt upgrade -y && apt autoremove -y

    apt install -y strongswan strongswan-swanctl strongswan-pki \
libstrongswan libstrongswan-standard-plugins libstrongswan-extra-plugins \
libcharon-extra-plugins libcharon-extauth-plugins charon-systemd \
libtss2-tcti-tabrmd0

5. Obtain server certificate

    apt install -y snapd

    snap install --classic certbot

    ln -s /snap/bin/certbot /usr/bin/certbot

    certbot certonly --standalone --agree-tos \
--register-unsafely-without-email --key-type rsa --rsa-key-size 2048 \
-d vpntest3.mydomain.mytld

    certbot renew --dry-run

6. Expand file permissions to allow strongSwan access

    cd /etc/letsencrypt

    find . -type d -exec chmod 755 {} \;

    find . -type f -exec chmod 644 {} \;

    chmod 755 /etc/swanctl/private
Edit /etc/apparmor.d/usr.sbin.swanctl: 
    vi /etc/apparmor.d/usr.sbin.swanctl
Add a line after the swanctl line: 
    # All reading configuration, certificate, and key files beneath /etc/swanctl/
    /etc/swanctl/**               r,
    /etc/letsencrypt/**           r
Save the file. Reload AppArmor: 
    systemctl reload apparmor

7. Symlink to Let's Encrypt certificates

    ln -s /etc/letsencrypt/live/vpntest3.mydomain.mytld/privkey.pem /etc/swanctl/private/privkey.pem

    ln -s /etc/letsencrypt/live/vpntest3.mydomain.mytld/chain.pem /etc/swanctl/x509ca/chain.pem

    ln -s /etc/letsencrypt/live/vpntest3.mydomain.mytld/cert.pem /etc/swanctl/x509/cert.pem
Create a script in /etc/letsencrypt/renewal-hooks/post: 
    vi /etc/letsencrypt/renewal-hooks/post/loadall.sh
Add this script: 
    #!/bin/sh
    /usr/sbin/swanctl --load-all

Save the script file.

8. Generate random user password

    < /dev/urandom tr -dc a-z0-9 | head -c${1:-16}; echo ''
Example: 
    wm77bw6inf77dz48

9. Create strongSwan configuration

Create your swanctl configuration file /etc/swanctl/swanctl.conf using the recommended VICI format:

    vi /etc/swanctl/swanctl.conf

Replace the existing contents. Use this as a template:

    connections {
        rw {
            version = 2
            pools = rw-pool-v4, rw-pool-v6
            local {
                auth = pubkey
                certs = cert.pem
                id = vpntest3.mydomain.mytld
            }
            send_cert = always
            remote {
                auth = eap-mschapv2
                eap_id = %any
            }
            children {
                rw-child {
                    local_ts = 0.0.0.0/0,::/0
                }
            }
        }
    }
    pools {
        rw-pool-v4 {
            addrs = 10.3.0.0/16
            dns = 1.1.1.1, 1.0.0.1
        }
        rw-pool-v6 {
            addrs = fec3::/120
            dns = 2606:4700:4700::1111, 2606:4700:4700::1001
       }
    }
    secrets {
        eap-emily3 {
            id = emily3
            secret = wm77bw6inf77dz48
        }
    }

Save the swanctl configuration file.

10. Run strongSwan on server

Stop and disable strongswan-starter:

    systemctl stop strongswan-starter

    systemctl disable strongswan-starter

Restart strongSwan:

    systemctl restart strongswan

    systemctl status strongswan

    journalctl -u strongswan

    exit

11. Configure iOS client

Open Settings > General > VPN & Device Management > VPN.

Tap Add VPN Configuration.

  • Type = IKEv2
  • Description = vpntest3.mydomain.mytld
  • Server = vpntest3.mydomain.mytld
  • Remote ID = vpntest3.mydomain.mytld
  • Local ID = blank
  • User authentication = Username
  • Username = emily3
  • Password = wm77bw6inf77dz48

Tap Done.
